# Agents Orchestration

Каждый MCP-сервер — это специализированный субагент. Используй их **до** того как начнёшь писать код.

## Когда какой агент звать

- **`mcdev`** → нужен исходник Minecraft, поиск метода/класса, граф вызовов
  Примеры: «как vanilla обрабатывает X», «кто вызывает `LivingEntity.hurt`», «что в `BowItem.use`»

- **`context7`** → актуальная документация Fabric / NeoForge / Gradle / Java-либ
  Триггер: незнакомый API loader-а, новые версии, мажорные обновления

- **`github`** → найти **рабочий пример** в реальных open-source модах
  Триггер: «как другие сделали X», поиск use-case паттернов

- **`fetch`** → конкретная статья/wiki/issue по URL

- **`sequential-thinking`** → задача разбивается на >3 нетривиальных шага, или есть ветвления и зависимости. Маленькие задачи — без него (это лишний overhead).

- **`memory`** → запомнить решения по проекту между сессиями (название мода, выбранные ID, архитектурные решения)

- **`filesystem`** → чтение/запись файлов проекта (это базовый, не специализированный)

## Pipeline для типовых задач

### Новая фича
1. `sequential-thinking` — план если фича сложная
2. `mcdev` — посмотреть как vanilla делает похожее
3. `context7` — проверить актуальный API loader-а
4. `github` — глянуть примеры в чужих модах если нужно
5. Писать код
6. `memory` — сохранить ключевые решения

### Исправление бага / краша
1. Прочитать stacktrace (filesystem)
2. `mcdev` — посмотреть исходник класса в стеке
3. `mcdev` callgraph — кто вызывает / что вызывает
4. Фикс точечный, без рефакторинга

### Совместимость с другим модом
1. `github` — найти исходник этого мода
2. `mcdev` — общие точки в Minecraft API
3. Решение: mixin, события, или soft-dep

## Правила оркестрации

- **Сначала запрос к агенту, потом код.** Не угадывать API по памяти, особенно для последних версий MC.
- **Один агент за раз** для последовательных шагов; параллельно — только если запросы независимы.
- **Не звать всех подряд** — каждый вызов = токены. Если задача тривиальная (опечатка, переименование) — пиши сразу.
- **`sequential-thinking` не для болтовни** — только для реально многошаговых задач.
- **Результаты `mcdev` / `context7` краткой выжимкой** в ответе, не копировать весь класс целиком.
