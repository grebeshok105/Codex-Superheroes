# План: экспериментальный Скорпион на шейдерах (Veil + Satin)

Цель: новый герой **Скорпион (Mortal Kombat, C-тир)** сразу в «экспериментальном» виде —
с шейдерным визуалом, и заодно обкатка рендер-либ для глобального апгрейда визуала всего мода.

---

## 1. Факты по либам (проверено по Modrinth, 2026-06)

| Либа | Актуальная версия | Поддержка MC | Подходит нам? |
|---|---|---|---|
| **Veil** (Moonflower/Foundry) | 4.1.2 | **только 1.21.1** (Fabric/NeoForge) | Да, но надо поднять мод с 1.21.0 на **1.21.1** |
| **Satin** (Ladysnake) | 2.0.0 | **1.21 – 1.21.1** Fabric | Да, работает уже сейчас |
| Satin 3.0.0-alpha | 3.0.0-alpha.1 | 1.21.2–1.21.4 | Нет (не наша версия) |

Ключевой вывод: **под чистую 1.21.0 сборок Veil не существует.**
Обязательный шаг — бамп мода на **1.21.1** (для 1.21.0↔1.21.1 маппинги и Fabric API совместимы,
ожидается лёгкий бамп `gradle.properties` без правок кода).

Роли либ:
- **Veil** — тяжёлый рендер-фреймворк: своя партикл-система (квады/трейлы/glow), пост-пайплайн
  (bloom, кастомные post-шейдеры c `#include`-препроцессором), deferred-освещение, динамические
  точечные света, дебаг-редактор шейдеров прямо в игре (горячая перезагрузка).
- **Satin** — лёгкая обёртка над ванильными post-process шейдерами: экранные эффекты
  (искажение, виньетка, цветокор) через managed `ShaderEffect` + события рендера.

Стратегия: **Veil — основной**, Satin 2.0.0 — запасной/дополнительный для чисто экранных
эффектов, если что-то из Veil-пайплайна окажется сырым.

## 2. Скорпион — геймплейный слой (уже написан, ветка `viktor/scorpion-hero`)

- **Тир C**, без полёта. HP 24, броня **14 + 6 прочности**, +3 атака, +10% скорость, КБ-рез 0.3.
- Ресурс: «Адское пламя» (150, реген 1.2/тик).
- Пассивка «Дитя Преисподней»: полный иммунитет к огню/лаве + жгучая аура (враги в 2.6 блока загораются).
- 4 обычные способности (без ульты):
  1. **Адский гарпун** («GET OVER HERE!») — кд 9с: хитскан-конус 22 блока, 8 урона, поджог,
     притяжка цели К Скорпиону (паттерн моргенштерна Рем, скорость до 1.35 б/тик).
  2. **Извержение Преисподней** — кд 10с: столб огня в точке прицела (26 блоков), 10 урона,
     поджог 5с, подброс врагов.
  3. **Адский телепорт** («Toasty!») — кд 7с: блинк за спину цели (6 урона + поджог) или
     в точку прицела до 28 блоков.
  4. **Дыхание Преисподней** — кд 30с: огнемётный конус 9 блоков, 2.5 секунды канала,
     2 урона каждые 4 тика + поджог.

## 3. Скорпион — шейдерный слой (Veil, клиент)

Через интерфейс `ScorpionFx` с двумя реализациями (graceful degradation):
`VanillaScorpionFx` (текущие партиклы) и `VeilScorpionFx` (включается, если Veil загружен).

| Эффект | Чем делаем |
|---|---|
| Hellfire-партиклы (огонь с «угольками» и glow, свои текстуры) | Veil quad/trail particles + emissive |
| **Bloom** на всём огне Скорпиона | Veil post-pipeline (bloom) |
| **Heat haze** (марево) вокруг столба огня и при дыхании | Veil post-шейдер (UV-дисторшн по маске) |
| Огненная **цепь гарпуна** как светящийся трейл, а не точки | Veil trail renderer |
| Динамический **точечный свет** на гарпуне/телепорте/дыхании | Veil dynamic lights |
| «Адская виньетка» на весь экран во время Дыхания Преисподней | Veil post (fallback: Satin 2.0.0) |

## 4. Этапы работ

1. **Бамп 1.21.0 → 1.21.1** (`gradle.properties`: minecraft, fabric-api, yarn/mojmap;
   полная сборка + smoke-тест) — отдельный коммит.
2. Подключение Veil 4.1.2 (maven Modrinth/blamejared) как зависимость + запуск пустого
   Veil-клиент-слоя; проверка, что мод грузится и FPS не падает.
3. Мердж геймплейного слоя Скорпиона (`viktor/scorpion-hero`): код, скин (проверен — wide,
   без дырок), предмет «Кунай Скорпиона», 4 иконки способностей, lang ru/en, HUD-тема.
4. `VeilScorpionFx`: партиклы → трейл цепи → bloom → heat haze → виньетка дыхания
   (в этом порядке, каждый шаг проверяемый).
5. Тест вдвоём на сервере, тюнинг яркости/производительности.
6. Решение по **глобальному визуалу**: если Veil зашёл — план №2 (bloom на лазеры
   Хоумлендера/камехамеху, динамический свет на энергоабилки, пост-эффекты ультов).

## 5. Риски

- Veil — бета: документация неполная (GitHub wiki + примеры из исходников), API может меняться.
- Бамп на 1.21.1: теоретически могут вылезти мелкие несовместимости (правятся на месте).
- Veil становится **обязательной зависимостью** для шейдерной жарки. Через `ScorpionFx`-абстракцию
  держим возможность собрать и «без-Veil» жарку, если понадобится раздать кому-то ещё.
- Производительность: bloom/deferred на слабых ПК — добавим конфиг-флаг отключения FX.
