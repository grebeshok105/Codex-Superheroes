## 0. Контекст и исследование (что уже выяснено)

### 0.1 Наш мод (Codex-Superheroes, MC 1.21.1 Fabric, Java 21)
- Герой описывается интерфейсом `hero/Hero.java`: `getId`, `getAbilities()`, `getDimensions(Pose)` (хитбокс), `getSkinTexture()` (оверрайд скина на ванильную модель игрока), `applyPassives/removePassives`, `getTheme/getHudConfig`.
- `DoctorStrangeHero` — намеренно «тонкая оболочка»: id `superheroes:doctor_strange`, скин `textures/entity/hero/doctor_strange.png`, способности `MIRROR_DIMENSION` + `MIRROR_MODE_CYCLE`. Весь бюджет — в способности Зеркального измерения (см. PR #31).
- Скин/модель: оверрайд текстуры через client-mixin `AbstractClientPlayerSkinMixin` + `PlayerRendererMixin`. Хитбокс — через `PlayerDimensionsMixin` + `Hero.getDimensions`. Поза/полёт — `PlayerFlightPoseMixin`, `LocalPlayerFlightMixin`, `CameraMixin`.
- Уже есть инфраструктура «глитч-текста»: `client/hud/ClientHudGlitch.java` (`obfuscateChance()`), `GuiVanillaGlitchMixin`, `ChatComponentMixin`, `GuiOverlayMessageMixin`, `MadnessHudOverlay` — основа для эффекта шифрования (п. 11).
- Зеркальное измерение (PR #31): server `MirrorDimensionController` (зона-ловушка, циклы режимов), client `MirrorWarpFlashHud` (чёрная вспышка), payload `MirrorDimensionS2CPayload` (ON/OFF/KEEPALIVE/SWITCH), `IrisShaderBridge`.
- Эмодзи Железного Человека (п. 6): `HeroInfoPanelHud.drawIronManStatus()` строка `EmojiIcons.draw(g, HudIcons.PassiveGlyph.REACTOR, …)` — рисует глиф «реактора», который системный шрифт показывает как фиолетовый «гексагон»-эмодзи. **Это и есть тот эмодзи.**

### 0.2 Мод друга (`falbiks_heroes` 1.0.0, "Falbiks Heroes")
Чистая CCA-архитектура (Cardinal Components / ladysnake) — идеально для адаптивной кражи:
- `core/component/ModComponents`: `ComponentKey<HeroComponent> HERO`, `ComponentKey<ResourceComponent> RESOURCES` (на сущностях-игроках).
- `core/component/HeroComponent`: хранит `heroId`, кулдауны, активные флаги. Методы: `getHero()`, `setHero(id)`, `clearHero()`, **`boolean tryActivate(AbilitySlot)`** (server-side точка входа ЛЮБОЙ активной способности), `serverTick()` (тики/пассивки), `deactivate*`.
- `core/hero/Hero` (`id`, `EnumMap<AbilitySlot,Ability>`), `HeroRegistry.getAll()`, `core/ability/Ability` (`onActivate/onTick/onDeactivate`, `canActivate`), `AbilityRegistry`.
- Пассивки: атрибут-модификаторы через `HeroPassives.syncModifier(...)` + per-tick mixin'ы (`LivingEntityTickMixin`, `OutgoingDamageMixin`, `PlayerDamageMixin`, `ItemUseSpeedMixin`, `SpeedJumpMixin`, и т.п.).
- Герои сейчас: Quicksilver, Strange, Graviton, Dr. Manhattan, Aizen, Saitama, Thanos, Levi, Madara, Twice, Subaru, Ant-Man, Garou, Sung Jin-Woo + босс Grebeshok. **Мод обновляется → новые герои появятся.**

**Ключевой вывод для адаптивности:** все активные способности любого героя (нынешнего и будущего) проходят через ОДИН чокпоинт — `HeroComponent.tryActivate`. Все пассивки/тики — через `serverTick` + общие mixin'ы. Значит, отнять способности «у всех и навсегда вперёд» можно, заглушив эти чокпоинты, **не зная конкретных героев**.

## 1. Архитектура решения (высокоуровнево)

Переименовываем/перепрофилируем `DoctorStrangeHero` → **`PandoraHero`** (id `superheroes:pandora`). «Зеркальное измерение» становится «Домом тщеславия» — расширенным режимом-ареной, в котором у Пандоры абсолютные полномочия. Вся новая логика делится на:

- **Server core:** `PandoraHero`, `HouseOfVanityController` (расширение `MirrorDimensionController`), `VanityAuthority` (полёт/иммунитет/выход за границы), `AbilityStripService` (кросс-мод отъём), `PandoraReviveHandler` (воскрешение за спиной убийцы).
- **Cross-mod bridge (адаптивный):** `falbiks` soft-integration через `@Pseudo`-миксины (target-by-string, не требует compile-dependency и не падает, если мода друга нет).
- **Client:** `PandoraSkin`/render-scale (75%), `VanityTextCipher` (прогрессивное шифрование ВСЕГО текста), HUD «Дома тщеславия», звук детского смешка.

## 2. Детализация по пунктам запроса

### П.1 — Зона-ловушка телепортит на поверхность, а не в блоки
- В `MirrorDimensionController.enforceZone()` при «yank back» сейчас тянем к центру по горизонтали, Y берём как есть → попадание в стену/землю.
- **Фикс:** перед телепортом находить безопасный Y: `level.getHeightmapPos(MOTION_BLOCKING_NO_LEAVES, targetXZ)` → ставим на верх поверхности; затем проверка `noCollision` для хитбокса игрока, при коллизии — подъём вверх по 1 блоку до свободного места (cap, напр. +6). Партиклы портала — на финальной позиции.
- Тот же безопасный-телепорт хелпер использовать при «вытягивании всех в радиусе 50» (п.2) и при выходе/входе.

### П.2 — «Дом тщеславия» затягивает ВСЕХ в радиусе 50 блоков от точки каста
- При активации (server, `onActivate`): собрать `level.getEntitiesOfClass(ServerPlayer, AABB вокруг caster, r=50)` минус сам caster (и, по решению — минус союзники/команда, см. вопрос Q2).
- Каждому добавить `Session` в контроллер (как victim), применить шейдер-варп (под чёрной вспышкой у каждого), посадить в зону-ловушку с центром = точка каста. Новые игроки, вошедшие позже в радиус, — опционально дозатягивать (вопрос Q2b).
- Мобов — по решению (вопрос Q2c). По умолчанию: только игроки.

### П.3 — Смена режима РАБОТАЕТ НА ВСЕХ ИГРОКАХ, не только на Стрэндже
- Сейчас `MirrorModeCycleAbility`/`cycleMode(caster)` могут неявно опираться на то, что caster — носитель измерения. **Жёсткая проверка:** ability действует на ВСЕХ текущих victim'ов сессии (по списку контроллера), независимо от того, какой герой у victim'а и кто он. Перебор `controller.getActiveSessions()` и `switchMode` каждому. Покрыть тестовым сценарием (мульти-игрок). Пользователь проверит вручную.

### П.4 — Замена Доктора Стрэнджа на Пандору
- **Скин:** приложен `pandora_skin.png` (64×64, **classic / 4px arms**). Кладём в `assets/superheroes/textures/entity/hero/pandora.png`. `PandoraHero.getSkinTexture()` → этот путь.
- **Руки 4px / wide:** скин классический (wide). Модель-оверрайд использует **wide-модель**. Для поддержки и slim — авто-детект по альфе зоны рукава (x≥54 прозрачна → slim) ИЛИ конфиг-флаг `pandora_slim`. (Вопрос Q1: достаточно ли wide, или нужен полноценный auto/both?)
- **Переименование:** новый id `superheroes:pandora`, lang-ключи ru/en, тема (HeroTheme) — фиолетово-золотая (Authority of Greed). Старый `doctor_strange` либо удаляем, либо оставляем алиас (вопрос Q4).

### П.10 — Абсолютные полномочия в «Доме тщеславия»
Пока «Дом тщеславия» активен, для Пандоры (`VanityAuthority`):
- **Полёт** (creative-style flight) + **иммунитет** к урону/эффектам.
- **Выход за границы измерения:** зона-ловушка (50 блоков) НЕ применяется к самой Пандоре — она может вылетать наружу, остальные заперты.
- **Отъём способностей** (в прямом смысле): у victim'ов отнимаются ВСЕ активные/пассивные способности (наши + falbiks), но **скин/костюм остаётся**. Реализация — `AbilityStripService` (см. §3).
- **Негативные эффекты** victim'ам: лёгкие «сводящие с ума» — Slowness, Mining Fatigue, периодически Nausea/Darkness слабого уровня, рандомные микро-дизориентации (вопрос Q6 — список/сила).

### П.11 — Прогрессивное шифрование ВСЕГО текста (по аналогии с Регулусом/Евангелием)
- Пока локальный игрок (victim) в «Доме тщеславия»: с каждой секундой растёт коэффициент `cipher∈[0..1]`. ВЕСЬ текст на экране (HUD, чат, оверлеи, кастомный шрифт ЖЧ, любые) постепенно заменяется на «шифр».
- **Чокпоинт:** глобальный mixin в рендер шрифта (`net.minecraft.client.gui.Font` draw / `GuiGraphics.drawString*`) — перехват ВСЕХ строк и поглифовая замена с вероятностью `cipher` на символы шифра (как `§k`-обфускация, но детерминированно нарастающая). Один хук покрывает и наш HUD, и кастомный шрифт ЖЧ, и HUD'ы falbiks (они в итоге рисуют через ванильный Font).
- Переиспользуем `ClientHudGlitch`/`MadnessHudOverlay` стиль символов. На выходе из измерения — мгновенно `cipher=0`, всё нормально.
- Вопрос Q3/Q6: шифруется ли текст у самой Пандоры (у неё авторитет — вероятно НЕТ) и у не-victim'ов вне измерения (НЕТ).

### П.12 — Кража способностей у героев falbiks + адаптивность + шифр для них
Самое технически важное. Требование: работать на ВСЕХ falbiks-героях, включая будущих, без хардкода по именам.

**Подход — soft-зависимость через `@Pseudo`-миксины (target-by-string), мод друга в classpath в рантайме, но НЕ compile-time зависимость:**
1. Добавить в наши mixin-конфиги `@Pseudo @Mixin(targets="com.falbiks.heroes.core.component.HeroComponent")`:
   - `@Inject(method="tryActivate", at=HEAD, cancellable=true)` → если игрок помечен `vanityStripped`, вернуть `false` (блок ЛЮБОЙ активной способности — текущих и будущих героев).
   - `@Inject(method="serverTick", at=HEAD, cancellable=true)` → если `vanityStripped`, пропустить тик (глушим пассивки/проки, не трогая `heroId` → костюм/скин остаётся).
2. Доп. подавление атрибут-пассивок: пока `vanityStripped`, каждую серверную секунду снимать у victim'а falbiks-атрибут-модификаторы (их id известны как namespace `falbiks_heroes:*`) — generic-проход по `AttributeMap`, удаляя модификаторы с этим namespace (адаптивно к новым).
3. Флаг `vanityStripped` — наш data-attachment на игроке (Fabric Attachment API), ставится `AbilityStripService` при входе в «Дом тщеславия» и снимается на выходе → способности возвращаются.
4. **Наши** герои (Codex-Superheroes) глушатся аналогично — в нашем `AbilityService`/активации добавить проверку `vanityStripped` (блок activate + skip applyPassives) при сохранении скина.
5. **Шифр (п.11) для falbiks-игроков** работает автоматически — он клиентский и глобальный (на victim'е), не зависит от того, из какого мода жертва.

**Почему адаптивно:** мы цепляемся за родовые чокпоинты (`tryActivate`/`serverTick`/namespace атрибутов), а не за конкретных героев. Новый герой в falbiks-апдейте автоматически проходит те же чокпоинты → автоматически обесточивается. `@Pseudo` + `required=false` гарантирует, что без мода друга наш мод не падает.

**Риски/нюансы:**
- Имена методов/сигнатуры (`tryActivate`, `serverTick`) могут поменяться в будущих версиях falbiks → миксин «мягкий» (required=false) + лог-предупреждение, плюс fallback через рефлексию `clearHero()`/восстановление. (Вопрос Q8b: ок ли fallback с полным clearHero, если скин на секунду слетит?)
- Obfuscated vs dev mappings: в рантайме (Yarn intermediary) target-by-string по нормальному имени класса falbiks работает, т.к. это его собственные классы (не ванильные). Refmap не нужен для не-ванильных таргетов.

## 6. УТОЧНЕНИЯ ОТ ПОЛЬЗОВАТЕЛЯ (2026-06-13, ответы на Q1–Q9) — ФИНАЛ

- **Q1:** Только **wide** модель (slim не нужен). Оверрайд скина на ванильной модели.
- **Q2:** Затягивать **ВСЕХ** игроков в радиусе 50 (без исключений — союзников тоже; «похуй»). Мобов — не обязательно (фокус на игроках).
- **Q3:** Стрип способностей и шифр — **только пока жертва в Доме тщеславия**, на выходе всё возвращается. ✓
- **Q4:** **Полностью заменить** Доктора Стрэнджа на Пандору; **способности Стрэнджа тоже переходят Пандоре** (Дом тщеславия + Каприз творца — это и есть его способности, переименованные).
- **Q5:** Имена ок. ИЗМЕНЕНИЕ набора способностей:
  - Убрать «Кукольные нити» (pull). Вместо неё — **шейдерные верёвки** (`SpatialBindAbility`): реальные анимированные тросы, прибитые в небо/землю/здания вокруг цели, плавно стягивают и **полностью обездвиживают** (root). Чисто визуально через шейдер/рендер + сервер-side root.
  - Новая **«Сжатие пространства»** (`SpaceCrushAbility`): сжимает связанную (или любую) цель — урон/добивание, эффект схлопывания пространства. **Только по активации.**
  - **ВСЕ способности, работающие только в измерении, ПОЯВЛЯЮТСЯ В UI/доступны ТОЛЬКО когда Пандора в Доме тщеславия.** Вне дома их нет. Сжатие — только при активации.
- **Q6:** «Сводящие с ума» эффекты = **это и есть прогрессивное шифрование текста** (п.11). (Доп. лёгкие зелья-эффекты опционально, но ядро — шифр.)
- **Q7 (КИНЕМАТОГРАФИЧЕСКАЯ СМЕРТЬ — детально):** без лимита воскрешений, **инвентарь сохраняется**. Последовательность при гибели Пандоры:
  1. **Остановка времени:** реально стоп — камера не двигается, ввод заблокирован, мобы/сущности замирают.
  2. **Перекрас мира:** плавно (smooth fade-in) весь экран → **красный градиент как на референсе** (`docs/plans/assets/pandora_death_ref.png`); весь ландшафт/мобы/сущности — **сплошные чёрные силуэты с чёрной обводкой**; **скрыть ВЕСЬ HUD, ники, всё** (голый экран как на F1).
  3. Пандора **поднимается в воздух**, вокруг — эффекты крови и пр. (имплементацию деталей доверяет мне).
  4. Через пару секунд эффект **плавно спадает**, мир возвращается.
  5. Пандора **появляется за спиной убийцы** + **ехидный смешок** (звук).
  6. После анимации — **неуязвимость** (урон по ней не проходит = скрытый хитбокс). → реализую как **окно i-frames ~5s** (иначе её нельзя убить и катсцена не повторится; длительность уточняема).
  7. Если убийца был **отвёрнут** (Пандора вне его поля зрения) — его камеру **плавно разворачивает к Пандоре**; если она была в поле зрения — разворота нет.
  - Эффект приходит ПЛАВНО, «закрашивает весь мир». Реализация: клиентский **пост-процесс шейдер** (силуэт по depth/edge + красный градиент) + блокировка ввода/камеры + заморозка тиков сущностей на сервере (или клиентский freeze-overlay).
- **Q8:** Текст у самой Пандоры **НЕ шифруется**. Кража способностей — **через fallback ок** (`clearHero` и т.п.), главное — **скин остаётся**.
- **Q9:** Тестирует на **реальном мультиплеере**.

### Технический план кат-сцены смерти (`PandoraDeathCinematic`)
- Триггер: перехват летального урона у игрока-Пандоры (`LivingEntityFallDamageMixin`-стиль hook на `hurt`/`ALLOW_DEATH`), отмена смерти → запуск катсцены вместо неё.
- Сервер: рассылает S2C payload `PandoraDeathCinematicS2CPayload` всем зрителям в радиусе (или хотя бы убийце+близким): фазы (FADE_IN/HOLD/FADE_OUT), позиция Пандоры, id убийцы. Сервер замораживает соответствующие сущности на длительность.
- Клиент: `DeathCinematicOverlay` (full-screen) + пост-шейдер `pandora_silhouette.fsh`/`.json` (Veil или vanilla post chain) — рендер сцены в чёрный силуэт + красный градиент-фон; `MinecraftClient` input lock (как pause без меню), скрытие HUD (`InGameHud` hidden), скрытие nameplate'ов. Плавный alpha по фазам.
- После HOLD: телепорт Пандоры за спину убийцы (server), звук `child_giggle`, форс-разворот камеры убийцы (если был отвёрнут) через client lerp; затем `vanityInvulnerable` attachment на ~5s.
