# Invincible v3.13.0 Implementation Plan

> For agentic workers: implement this plan task-by-task and keep the mod buildable after each code slice.

**Goal:** Add Invincible as a new transformable hero with supplied skin, suit item, flight, durable passives, and two impact-focused combat abilities.

**Architecture:** Use the existing generic hero pipeline: `Hero` implementation, `Heroes` registry, `TransformationItem`, `ModItems`, creative tab, lang keys, item model, and runtime textures. Avoid new packets, HUD overlays, custom damage types, or client renderers; existing skin sync, ability routing, resources, cooldowns, and flight mixin are enough for this release.

**Tech Stack:** Fabric 1.21, Java 21, Gradle/Loom, Minecraft resource JSON, PNG assets.

## Tasks

1. Add Invincible attributes: armor, toughness, attack damage, speed, max health, knockback resistance, attack speed, jump strength, and step height.
2. Add `InvincibleHero` with human dimensions, high durability passives, fall damage immunity, landing shockwave, yellow-blue HUD theme, and abilities `flight`, `viltrumite_charge`, `meteor_slam`.
3. Register the hero in `Heroes`.
4. Add `viltrumite_charge`: active forward rush with server tick collision, cooldown, damage, knockback, dust/crit particles, and no custom damage type.
5. Add `meteor_slam`: active AoE impact around the player, stronger when airborne, with shockwave, particles, screen shake, cooldown, and energy cost.
6. Register new ability IDs and ability instances.
7. Add `InvincibleSuitItem`, item registry entry, creative tab entry, item model, skin texture, generated item icon, and localization.
8. Add passive tooltip count to `AbilityDescriptions`.
9. Update project docs for version `3.13.0` and 14 registered heroes.
10. Run `cmd /c ".\gradlew.bat build --no-daemon -x test --console=plain"` and confirm `build/libs/superheroes-3.13.0.jar`.
11. Review `git diff`, commit with `feat(invincible): add hero`, and push the branch to `origin`.
