# v3.17.0 — Реворк системы ударов: тиры, пробивание стен телом, ядро удара, новый HUD

**Цель:** переработать тировую систему атак (ЛКМ/ПКМ), сделать пробивание стен ТЕЛОМ персонажа (с инерцией и уроном за каждую стену), запретить ударам ломать блоки, переделать индикатор заряда у прицела, ускорить заряд (Tier 2 = 1.5с, Tier 3 = 3с).

**Архитектура:** всё считает единое ядро `physics/` — `CombatImpactEngine` (вектор, сила, урон, множители героя) расширяется, а полёт тела сквозь стены выносится в новый серверный трекер `BallisticBodyTracker`: любое «запущенное» тело (жертва удара tier 2/3, сверхзвуковой полёт) регистрируется и каждый тик просчитывается его путь — блоки на пути ломаются по `BlockBreakPolicy`, скорость частично сохраняется (инерция → несколько стен), урон за стены суммируется в реальном времени.

**Ветка:** `viktor/v3.17.0-impact-rework` поверх `viktor/v3.16.3-user-fixes` (PR #7). Версия: `3.16.3 → 3.17.0` (minor: новая система, по version-bump-policy).

## Задача 1. Тайминги заряда

`ImpactChargeRules`: `TIER_2_TICKS = 30`, `TIER_3_TICKS = 60`, `CAP_TICKS = 80`.
Клиент уже шлёт тики и капит по `CAP_TICKS` — менять только константы. Проверка: зажать ПКМ, Tier 2 через 1.5с, Tier 3 через 3с.

## Задача 2. Удары больше НЕ ломают блоки

В `CombatImpactEngine.applyMeleeImpact()` удалить вызов `breakImpactBlocks` и сами методы `breakImpactBlocks`/`breakSphere` (перенос логики посферного лома — в `BallisticBodyTracker`, см. задачу 4). Из `ImpactProfile` убрать `breakBlocks/breakRadius/maxBlocks/maxHardness` (maxHardness переезжает в launch-параметры). Дебрис от самого удара оставить только как частицы (без лома).

## Задача 3. Тиры

### Tier 1 — ЛКМ (обычная атака)
Ванильный урон как сейчас (PASS), но в `HeroMeleeImpactController`:
- через `ALLOW_DAMAGE` + отложенный `END_SERVER_TICK` (AFTER_DAMAGE в Fabric 0.102 нет) добавить жертве небольшой доп. толчок: `+0.32 * heroPower` горизонтально, вверх ≤ 0.12;
- FX: 6–10 частиц стиля героя + лёгкая тряска экрана у атакующего и жертвы (`ScreenShakeS2CPayload`, intensity 0.25, 6 тиков) + ванильный звук.

### Tier 2 — ПКМ 1.5с
- урон `base * 1.8 * heroPower * weaponBias`, откидывание `2.6 * heroPower` гориз., вверх ≤ 0.4;
- VFX заметно жирнее: 30+ частиц, EXPLOSION, тряска 18 тиков, звук NETHERITE_HIT;
- жертва регистрируется в `BallisticBodyTracker` с `launchPower ≈ 35` (хватит на ~1–2 тонкие стены).

### Tier 3 — ПКМ 3с, ДИНАМИЧЕСКАЯ
- базово: урон `base * 3.0 * heroPower * weaponBias`, откидывание `4.5 * heroPower`, вверх ≤ 0.5;
- **динамика до 40%:** на каждый удар roll `variance ∈ [0.7 … 1.4]` (т.е. разброс до +40%), множится на урон, откидывание и интенсивность FX → каждый удар ощущается по-разному;
- случайный FX-вариант из пула стиля (3 варианта на стиль: кольцевая ударная волна / сноп искр+молний / взрыв дебриса), громкая связка звуков, тряска 28 тиков;
- жертва регистрируется с `launchPower ≈ 90 * heroPower * variance` (пробивает несколько стен).

## Задача 4. Ядро «тело-снаряд» — BallisticBodyTracker (главное)

Новый класс `physics/BallisticBodyTracker`:
- `launch(LivingEntity body, Vec3 velocity, double power, @Nullable ServerPlayer source)` — регистрация (Map по entityId: power-бюджет, источник, накопленный урон, тики жизни ≤ 60);
- тик в `END_SERVER_TICK`: для каждого тела:
  1. скорость < 0.55/тик или бюджет ≤ 0 → дерегистрация (дальше обычные коллизии);
  2. свип пути: AABB тела, раздутый на 0.35, продвигается от текущей позиции до `pos + velocity` шагами 0.5 блока;
  3. блоки в свипе, проходящие `BlockBreakPolicy.canImpactBreak` (maxHardness 12) → `destroyBlock(drop=false)`; за КАЖДЫЙ сломанный слой стены: `power -= 6 + hardness`, скорость `*= 0.78` (инерция — пробивает насквозь и летит дальше), **телу урон `2.0 + 0.6 * hardness` (generic, суммируется в реальном времени за каждую стену)**;
  4. непробиваемый блок на пути → жёсткая остановка: остаток скорости в урон (`speed * 4`, как краш), дерегистрация;
  5. VFX за пробитие: `WallImpactDebrisS2CPayload` (ids сломанных стейтов) + BlockParticleOption осколки самих блоков («кирпичики», ванильные — никаких розовых квадратов) + LARGE_SMOKE/CLOUD (дым/порох) + звук GENERIC_EXPLODE 0.6 + тряска поблизости.
- Источники запуска: melee tier 2/3 (жертва) и `FlightController` сверхзвук (сам игрок, power 50, обновляется пока летит). Удары руками блоки НЕ ломают — только тело, влетающее в стену.
