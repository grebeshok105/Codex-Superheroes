## §1. Уран-индикатор на HUD у игрока-Хоумлендера

**Цель:** когда поблизости в одном мире есть игрок с уран-кинжалом, у самого Хоумлендера на клиенте видно «угрозу» (значок радиации + зеленоватый тинт HUD).

### Что уже есть
- `UraniumDefenseController` (server) каждые 20 тиков сканирует игроков, шлёт `UraniumPressureS2CPayload(List<UUID>)` со списком pressured Хоумлендеров.
- `ClientUraniumPressureState.isPressured(UUID)` доступен на клиенте.

### Что нужно сделать
1. **Расширить S2C-протокол:**
   - Заменить `UraniumPressureS2CPayload` или добавить второй payload `UraniumThreatS2CPayload(boolean self, int sourceCount)` — клиенту получателя достаточно знать, является ли он сам pressured Хоумлендером и сколько носителей кинжала рядом.
   - Альтернатива: оставить существующий payload и на клиенте проверять `ClientUraniumPressureState.isPressured(localPlayerUuid)` в HUD-рендерах.

2. **Радиационный значок:**
   - Создать `UraniumThreatHud` (клиент) — рендерится только когда локальный игрок = Хоумлендер и `isPressured(self)`.
   - Текстура: иконка радиации (3 сектора). Если в `art-source/` нет — **спросить пользователя**, рисовать новую или взять из vanilla (нет vanilla радиации) → нужен PNG от пользователя или генерация процедурно через `GuiGraphics.fill` (3 круга и точка).
   - Позиция: над хотбаром, по центру или в правом верхнем углу. **Уточнить у пользователя.**
   - Пульсация: альфа осцилирует (sin) для привлечения внимания.

3. **Зелёный tint HUD:**
   - Добавить полупрозрачный зелёный fill (~`0x2000FF00`) поверх ResourceBar/AbilitiesTooltip когда self-pressured.
   - Можно реализовать через `ClientHudGlitch.uraniumTint()` (новый метод) — возвращает доп. tint-color, который применяется в `ResourceBarHud.renderInner` и `AbilitiesTooltipHud.renderInner`.

4. **Звуковой/визуальный фидбэк на entry:**
   - При первом моменте, когда self становится pressured — короткий звук «сигнал тревоги» (vanilla `block.beacon.activate` или новый OGG, **уточнить**) + ScreenShake intensity=0.4 длительность=10 тиков.

## §2. Eye Lasers — пульсирующий паттерн стрельбы

**Цель:** изменить непрерывный лазер на ритм «1 сек стрельба → 0.5 сек пауза → 2 сек стрельба → … → повтор» (паттерн пользователь записал именно так — после 2 сек цикл повторяется с 0.5 паузой; **уточнить у пользователя точную форму паттерна**).

### Что уже есть
- `EyeLasersAbility` — toggle ability. Каждый серверный тик пока активна → `fireBeam(player)`.

### Что нужно сделать
1. **Состояние пульсации:**
   - Добавить в `EyeLasersAbility` карту `Map<UUID, Integer> patternTickCounter` (или сохранить в `HeroData` как long-поле).
   - Каждый тик увеличиваем `patternTickCounter`. По модулю относительно общего цикла (1с + 0.5с + 2с = 3.5с = 70 тиков) определяем, активна ли стрельба.
   - Расчёт по фазе:
     - 0 ≤ tick < 20 (1с): стрельба
     - 20 ≤ tick < 30 (0.5с): пауза
     - 30 ≤ tick < 70 (2с): стрельба
     - tick ≥ 70: сбрасываем

2. **Визуальная пауза:**
   - Во время паузы `fireBeam` НЕ вызывается → урон не наносится, частицы не спавнятся, `ModNetworking.broadcastLaser` не шлётся → лазер визуально пропадает на клиентах.

3. **Звук:**
   - Каждый старт «активной» фазы (tick=0 или tick=30) → короткий sound cue (vanilla `entity.guardian.attack` или mod-sound, **уточнить**).

- Скорость полёта = vanilla creative-fly speed (через `flyingSpeed`).

   - В клиентском mixin/тике, пока полёт активен, выставлять `player.getAbilities().setFlyingSpeed(0.02f)` (vanilla = 0.05f). Или ещё медленнее — **уточнить степень**.
   - Опция: ограничить вертикальную скорость отдельно (через `setDeltaMovement`).

2. **Auto-disable каждые 5с:**
   - На сервере в `FlightController` (новый или существующий) для каждого игрока с активным flight-toggle вести счётчик тиков. На 100-м тике (5с):

     - Послать клиенту S2C-payload `FlightForceOffS2CPayload`.
     - Сбросить счётчик. При повторной активации — снова 5с.
   - **Уточнить:** засчитываем только время, когда игрок в воздухе и реально летит, или с момента активации toggle (даже если стоит на земле)?

3. **Нет восстановления маны во время паузы:**

### 5.1. Color drift к blood-black

В `AbilitiesTooltipHud` и `ResourceBarHud` все вызовы `theme.energyText()`, `theme.manaIcon()`, `theme.manaText()`, `theme.energyBar()` и т.д. — обернуть в `ClientHudGlitch.tintColor(...)`:
```java
int tinted = ClientHudGlitch.tintColor(theme.manaText());
g.drawString(..., tinted, ...);
```
Это даёт автоматический lerp к blood-black по ramp.

**Файлы:** `ResourceBarHud.java`, `AbilitiesTooltipHud.java` — пройтись по всем `theme.*Color()` вызовам и обернуть.

### 5.2. Glitch текста (`§k`)

При ramp ≥ 70% — для каждого `Component`-name/desc Регулуса в HUD:
- Добавить `.withStyle(ChatFormatting.OBFUSCATED)` с шансом `(ramp - 0.7) / 0.3`.
- Реализовать в `ClientHudGlitch.maybeObfuscate(Component) → Component`.
- Применить в `AbilitiesTooltipHud.drawAbilityRow` (имя+описание ВСЕХ способностей, не только counter_strike, когда self=Регулус и в безумии).
- Применить к ResourceBar числам (`String.valueOf(mana)`) — обернуть как `Component.literal(num)` с шансовым `§k`.

### 5.3. Jitter герба (badge)

В `AbilitiesTooltipHud.drawAbilityRow` есть рендер `badge` (значок типа способности). Сейчас он рисуется внутри `pose().translate(jitter, ...)` родительского рендер-вызова, поэтому *двигается со всем тултипом, но не индивидуально*.

Хочется: чтобы герб «дёргался независимо» — собственный jitter-offset с большей амплитудой (~3-4px). Реализация:
```java
g.pose().pushPose();
g.pose().translate(ClientHudGlitch.badgeJitterX(), ClientHudGlitch.badgeJitterY(), 0);
g.drawCenteredString(mc.font, badge, ..., ...);
g.pose().popPose();
```
Новый метод `ClientHudGlitch.badgeJitterX/Y()` возвращает `nextInt(-3..3) * ramp`.

### 5.4. Эффекты на vanilla UI (HP/food/hotbar)

Это требует mixin'а в `Gui` (vanilla). Подход:
- Новый `GuiHotbarMixin` / `GuiHealthMixin` — `@Inject(method = "renderPlayerHealth"/"renderHotbar", at = @At("HEAD"))` — push pose с jitter offset, `@Inject(at = @At("RETURN"))` — pop pose.
- Применять только если локальный игрок = Регулус и в безумии (`ClientHeroState.heroId() == REGULUS && ClientMadnessState.isMadness()`).
- Можно ограничить степень эффекта (jitter × 0.5 чтобы не ломать читаемость HP).

- F3 не трогаем (debug screen).

## §6. Iron Fists — резкий выпад по ЛКМ + бесконечная длительность

**Цель:** убрать авто-выпад/авто-агр. Включил способность → внутри неё игрок ХОДИТ ОБЫЧНО, но при ЛКМ по враждебной цели делает **резкий выпад** (рывок к цели + удар). Длительность не 10с, а пока сам не выключит. Цели для ЛКМ-выпада: только player/boss/hostile mob (не animals/villagers).

### Что есть
v1.0.36 переработал iron_fists в:
- Полное опустошение энергии при активации.
- 10-секундный фиксированный таймер.
- Авто-дэш к ближайшей цели в конусе 30 блоков.
- Лок других способностей и регена.
- Шоквейв на приземлении.

### Что нужно сделать (рефакт)

1. **Убрать таймер + лок:**
   - Удалить `IronFistsController` логику `lockTicks(player, 200)` и авто-выключение.
   - Toggle ability — отключается только когда игрок сам нажимает её ещё раз (или умирает).
   - Энергия:
     - **Уточнить:** жрёт ли всю энергию при активации, или каждый «выпад по ЛКМ» стоит фиксированного количества энергии (например 20)? **Пользователь не уточнил.**

2. **Убрать авто-дэш:**
   - Текущий `IronFistsController.tick` ищет ближайшую цель и дэшит. Удалить полностью.
   - Footsteps зацикленный звук — оставить или убрать? **Уточнить.**

3. **ЛКМ-выпад:**
   - Hook на `AttackEntityCallback` (Fabric API).
   - Условия: способность активна + цель `instanceof Player || HomelanderBossEntity || (Mob && monster && !animal)`.
   - Действие:
     - Резкий рывок: `player.setDeltaMovement(dirToTarget * 2.0, +0.4y, *)` — большая скорость, лёгкий «прыжок».
     - На контакте — `target.hurt(... , 8.0f)` + `setKnockback(target, 2.5)`.
     - Звук удара — текущий `homelander.iron_fists.impact`.
     - Шоквейв-эффект (vanilla `EXPLOSION` + radius dmg) — оставить.
   - Cooldown между ЛКМ-выпадами: ~5 тиков чтобы нельзя было заспамить.

4. **Никаких других целей:**
   - Animals (Cow, Pig, Sheep, …) — обычная атака, без выпада.
   - Villagers — без выпада.

- Пока на Хоумлендере висит эффект `MADNESS` (от молочка) — все эффекты урана отключены.

### §2 Eye Lasers — пульс-паттерн
- Конкретная форма паттерна на усмотрение Devin: цикл `1с стрельба → 0.5с пауза → 2с стрельба → 0.5с пауза → повтор`. Главное — выглядит «обрезанно».
- Применяется ТОЛЬКО к игроку-Хоумлендеру. Босс остаётся как есть.
- Существующая 50% защита от лазеров для целей с уран-кинжалом (`UraniumDefenseController.laserDamageMultiplier`) сохраняется без изменений.
- Во время эффекта `MADNESS` у Хоумлендера — пульс снимается (непрерывный лазер).
- Звук на старт активной фазы — короткий cue (vanilla `entity.guardian.attack`).

### §3 Полёт — медленный + auto-off каждые 5с
- Скорость на усмотрение Devin → 0.02 (vanilla 0.05).
- Счётчик 5с считаем ПОКА toggle активен (независимо от того, в воздухе или на земле).
- Cooldown между авто-выкл и возможностью включить снова: 1 сек.
- Применяется ТОЛЬКО к Хоумлендеру (Iron Man полёт остаётся как есть).
- При посадке на землю — авто-выкл срабатывает мгновенно (не ждём 5с).
- Во время `MADNESS` у Хоумлендера — нерф снимается (vanilla flight speed, без auto-off).

### §5 Регулус HUD glitch — фиксы
- Color drift цвет на усмотрение Devin → blood-black `0xFF330000`.
- Обфускация (`§k`) текста: только в безумии. Старт 10%, +10% каждые 15с, кап 80%. Счётчик ПЕРСИСТИТ через выходы из безумия (если такие будут) и сбрасывается ТОЛЬКО при смерти игрока.
- Vanilla UI глитчим: HP, food, hotbar (jitter pose). Чат НЕ трогаем. F3 НЕ трогаем.
- Cracks-overlay делаем в v1.0.38.
- Регулус: выход из безумия ТОЛЬКО через смерть. Существующие условия выхода (drain энергии до 0 / toggle off Lion Heart) не должны выключать madness — лишь смерть. (Lion Heart drain энергии остаётся, но madness не снимается).

### §6 Iron Fists — рефакт
- Энергозатраты: ПОЛНОЕ опустошение энергии при активации (как было в v1.0.36).
- Footsteps loop: оставляем.
- Шоквейв: на КАЖДЫЙ ЛКМ-удар, который реально достиг цель.
- Другие способности во время iron_fists: ЗАБЛОКИРОВАТЬ (как было).
- LMB-cooldown между выпадами: ≥40 тиков (минимум 2 секунды).
- Цели для ЛКМ-выпада: Player, HomelanderBossEntity, Monster (зомби, скелеты, криперы, рейдеры, фантомы…), а также Голем (`IronGolem`/`SnowGolem`). Animals и Villagers — НЕ выпад, обычная атака.
