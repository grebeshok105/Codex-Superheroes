# v3.34.0 — Железный человек: HUD, способности, Target-ESP

Статус: **черновик плана** (код не пишем, пока не согласуем). Версия после фикса орды — `v3.33.1`,
значит этот апдейт = `v3.34.0` (минорный, фич-пак Железного человека).

Связанные пункты задачи:
1. Освежить дизайн HUD Железного человека от первого лица (сейчас «страшные жёлтые полосы»).
2. Обновить способности: освежить репульсоры + добавить самонаводящиеся ракеты.
3. Target-ESP для Железного человека (на основе мода TargetESP, переписанный под нас).
4. (отдельно, уже сделано как проверка) текстуры Кадзухи/Скарамучи — руки 4px/wide.

## Что уже есть в проекте (контекст)

- **`JarvisOverlayHud`** — текущий HUD от 1-го лица. Рисуется голым `GuiGraphics.fill()`
  (плоские прямоугольники золотого цвета `0xFFFFC400`): уголки-скобки, боковые рельсы,
  баннер «J.A.R.V.I.S.», центральный прицел. Это и есть «жёлтые полосы».
- **`WildShaders`** — уже готовая GLSL-инфраструктура: SDF-шейдеры `hud_rect`, `hud_circle`,
  `hud_sector` (кольцевой сектор с мягкой границей + glow), `hud_icon_circle`. Их уже используют
  `AbilityBarHud`, `RadialMenuHud`, `HudUtil.neonPanel`, `MeleeChargeHud`. **HUD Джарвиса —
  единственный, кто их НЕ использует** → отсюда «дешёвый» вид.
- **`JarvisDetectionHud` + `IronManJarvisController`** — уже есть пайплайн обнаружения других
  игроков-героев: сервер сканит игроков, через 5 c шлёт `JarvisDetectionS2CPayload`
  (имя, heroId, дистанция, класс угрозы S/A/B/C/D, реплика Джарвиса), клиент проигрывает голос
  и показывает инфопанель. **Это идеальная база для пункта 3** (ESP на героев).
- **`RepulsorAbility`** — текущая атака: рейкаст на 40 блоков, 8 урона, лёгкий пуш,
  трассер через `RepulsorBeamRenderer` (WorldRenderEvents.AFTER_TRANSLUCENT).
- **`AbilityIds.IRON_MAN_SMART_MISSILE`** — id уже объявлен, но **способность не реализована и
  не привязана** к герою. Хорошая отправная точка для ракет.
- Способности Железного человека сейчас: FLIGHT, SUPERSONIC, REPULSOR, UNIBEAM, NANO_FORM,
  SUIT_SWITCH, LEGION.

## Пункт 1 — Редизайн HUD от первого лица (арк-реактор стиль)

**Цель:** заменить плоские жёлтые линии на «дорогой» интерфейс брони — мягкое свечение,
плавные дуги, аккуратная типографика, живые анимации. Цвет — фирменный hot-rod: золото
`#FFC400` + горячий красный `#E2342B` акцент + холодный голубой `#46D8FF` как «данные/реактор».

**Что рисуем (через `WildShaders`, не `fill`):**
- **Центральный прицел-реактор:** концентрические кольца (`hud_circle`/`hud_sector`) с лёгким
  вращением в разные стороны, тонкая «мушка» из 4 рисок, мягкий внутренний glow. Под огнём/при
  заряде — кольца сжимаются/краснеют.
- **Угловые модули (4 угла):** не «скобки-палки», а аккуратные L-рамки c засечками + мини-гекс
  + бегущая риска-сканер. Свечение и alpha-pulse через шейдер, без рваных пикселей.
- **Боковые дуги-телеметрия:** слева — энергия костюма (реактор), справа — целостность/нагрев,
  тонкими дуговыми барами (`hud_sector`) с делениями. Сейчас энергия рисуется отдельно — нужно
  свести стилистику.
- **Верхний баннер:** «STARK INDUSTRIES // J.A.R.V.I.S. ONLINE», тонкая разделительная линия с
  glow, маленький бегущий «typewriter»-эффект на статусной строке.
- **Нижний статус-тикер (опционально):** высота/скорость/режим полёта/высота над землёй —
  как бортовой компьютер.

**Файлы:** переписать `JarvisOverlayHud` целиком на `WildShaders`-вызовы (как `HeroInfoPanelHud`).
При необходимости — 1 новый шейдер `hud_scanline.fsh` (горизонтальный градиент-скан) и/или
`hud_ring.fsh` (кольцо с тиками), если существующих SDF не хватит. Палитра выносится в константы.

**Картинки-концепты:** перед кодом сгенерю 2–3 концепт-арта HUD (вид от 1 лица), покажу в Slack,
выберем направление.

## Пункт 2 — Способности: репульсоры + самонаводящиеся ракеты

### 2a. Освежение репульсоров (`RepulsorAbility`)
- **Заряд-режим:** короткое удержание = быстрый одиночный выстрел (как сейчас), длинное
  удержание = заряженный «бласт» (больше урона/радиус/отталкивание, knockback-конус).
- **Двойной репульсор:** визуально бьём с обеих ладоней (две трассы) при заряженном.
- **Лучше FX:** трассер с ядром+свечением (уже есть `RepulsorBeamRenderer` — усилим: толще,
  яркая сердцевина, дымные кольца, вспышка-impact, отметка на цели).
- **Звук:** многослойный «вуумп+бласт» вместо текущего blaze/beacon.
- Баланс: одиночный ~8 урона / 200 энергии (как сейчас); заряженный ~18–22 / ~450 энергии,
  мини-AoE в точке попадания. (Цифры под согласование.)

### 2b. Самонаводящиеся ракеты (новая `SmartMissileAbility` → `AbilityIds.IRON_MAN_SMART_MISSILE`)
- **Новая сущность-снаряд** `SmartMissileEntity` (на базе паттерна снарядов орды
  `HordeBombEntity`/проджектайлов): летит вперёд, затем плавно доворачивает (lerp вектора
  скорости) к захваченной цели; ограничение угла поворота, чтобы выглядело как настоящая
  самонаводка, а не телепорт.
- **Захват цели:** при активации берём цель из ESP/прицела (рейкаст + ближайший в конусе).
  Залп из N ракет (напр. 3) с веером и лёгкой задержкой; каждая берёт свою цель, если целей
  несколько, иначе все в одну.
- **Детонация:** взрыв-AoE без разрушения мира (как `ShockwaveUtil`/horde-бомбы), урон + пуш +
  частицы (дым/огонь/искры) + дымный шлейф в полёте.
- **Рендер:** `SmartMissileRenderer` — маленькая ракета (можно простая модель/айтем) + шлейф
  партиклами; на ESP-цели подсвечивается «LOCK».
- **Привязка:** добавить `IRON_MAN_SMART_MISSILE` в `IronManHero.getAbilities()` и на слот/радиалку.
- Баланс: залп 3×, ~6–8 урона на ракету + взрыв, кулдаун/стоимость энергии под согласование.

## Пункт 3 — Target-ESP Железного человека

**Идея:** Джарвис подсвечивает цели в мире (box/outline/обводка), а **на обнаруженных
игроков-героев накладывается усиленный ESP** с данными угрозы (используем уже готовый
`IronManJarvisController` + класс угрозы S/A/B/C/D).

> Лицензионно: код мода TargetESP не копируем (ARR/неизвестная лицензия). Делаем **свою**
> реализацию «по мотивам» (clean-room) — это и просил юзер («перепиши под нас»). Берём только
> идеи: стили, health-цвета, пресеты, плавные анимации.

**Рендер:** `WorldRenderEvents.AFTER_TRANSLUCENT` (как `RepulsorBeamRenderer`) — рисуем для
каждой видимой цели:
- **Режимы (стили):** `BOX` (3D-бокс по хитбоксу), `OUTLINE` (контур через OutlineBufferSource/
  glow-обводка), `CORNERS` (только углы бокса — «таргет-лок» в стиле брони), `TRACER` (линия от
  низа экрана к цели), `2D` (плоская рамка вокруг проекции). Вкл/выкл независимо.
- **Цвет по здоровью:** зелёный→жёлтый→красный по % HP цели (как в TargetESP).
- **Подписи:** имя, дистанция, HP-бар, у героев — класс угрозы [S/A/B/C/D] + иконка героя.
- **Усиление для героев:** на героя дополнительно — пульсирующая «LOCK»-рамка цвета класса
  угрозы (S = красный, A/B = оранж, C/D = жёлто-зелёный), маркер-ромб, доп.glow. Бьётся с
  голосовым докладом Джарвиса.
- **Анти-чит/чесность:** по умолчанию только видимые цели (рейкаст-проверка, не сквозь стены),
  чтобы оставалось «по-честному», но т.к. это синглплеер-шоукейс — сделаем тумблер «сквозь стены».
- **Настройки (GUI/конфиг):** включение ESP, набор активных режимов, дистанция скана, фильтры
  (игроки / мобы / только герои / только враждебные), пресет цветов (8 пресетов как у TargetESP +
  «Iron» по умолчанию), толщина/яркость/скорость анимации, «сквозь стены» on/off, лимит целей.
  Биндим на клавишу (как у мода — открытие меню) + тумблер ESP.

**Файлы (примерно):**
- `client/render/esp/TargetEspRenderer` — рендер боксов/обводок/трассеров.
- `client/esp/EspSettings` + экран настроек (или интеграция в существующий конфиг/радиалку).
- `client/esp/EspTargetScanner` — сбор видимых целей, health-цвет, фильтры.
- Связка с `JarvisDetectionHud`/`IronManJarvisController` для «герой → усиленный ESP».
- Клавиша в `ModKeys`.

## Пункт 4 — Текстуры Кадзухи и Скарамучи (проверка рук) — ГОТОВО

Проверил обе текстуры (64×64). **Обе нарисованы под SLIM-модель (Alex, руки 3px)**, а не под
wide/4px (Steve/classic). Сигнатура: колонки рук заполнены u=40–53 (14 колонок), u=54–55 пустые —
это раскладка slim. Для wide/4px-модели заполнены были бы все 16 колонок (до 55).

→ Если нужны под **4px/wide** — текстуры надо конвертировать (переразложить UV рук: front-грань
3px→4px, сдвинуть боковые/заднюю грани). Могу сделать автоматически и прислать готовые. Скажи,
какой вариант модели целевой (slim или wide).
