## 0. Контекст

Doomsday — DC Comics. Биоинженерный «совершенный» убийца, заточенный
под адаптацию: **умирает один раз от каждого источника урона; потом
этот источник на него больше не действует и его DPS сильно вырастает**.
Огромный, физически непропорционально сильный. Обычно изображается с
костяными шипами на руках/спине/груди, серой кожей, безглазой мордой.

В отличие от прошлых героев — у него нет «технологий» / «способностей
по кнопке» в чистом виде; вся его сила пассивная и завязана на
выживаемость.

## 2. Открытые вопросы (ОБЯЗАТЕЛЬНО спросить пользователя)

### Параметры персонажа

- **Q1. Размер.** Какой рост и ширина?
  - `getDimensions(Pose.STANDING)` — стандарт vanilla `width=0.6, height=1.8`.
  - Варианты: 1.5×3.0 (на голову выше), 2.0×4.0 (явно больше), 2.5×5.5 (мини-босс).
- **Q2. Шкала отрисовки.** Делать ли динамический scale модели
  (как планировалось для Хальксмена изначально), или просто увеличить
  hitbox + переотрисовать модель уже в нужном размере?
- **Q3. Запас HP.** Базовый HP бонус. Vanilla 20 = 10 сердец.
  - Скажем, +60 (всего 80 HP / 40 сердец) на «средний босс»
    или +200 (всего 220 / 110 сердец) на полу-непрошибаемого.
- **Q4. Броня и стойкость.**
  - `Attributes.ARMOR` — сколько (vanilla cap 30).
  - `Attributes.ARMOR_TOUGHNESS` — сколько.
  - `KNOCKBACK_RESISTANCE` — 1.0 (immune) или меньше.
- **Q5. Базовый урон.** `Attributes.ATTACK_DAMAGE` бонус.
- **Q6. Скорость.** `MOVEMENT_SPEED` бонус — % к vanilla.
- **Q7. Падение.** Cancel fall damage — да/нет.

### Уникальные механики

- **Q8. Адаптация (главная фича).** Реализуем механику
  «однажды убитый этим источником — иммун на всю оставшуюся жизнь
  трансформации»?
  - Если да — нужны вопросы:
    - **Q8.1.** Как определять «источник»: по типу `DamageSource`
      (fire / explosion / mobAttack / arrow и т.п.) или по
      конкретной сущности-атакующему?
    - **Q8.2.** Триггер «убит»: реальная смерть с уходом на экран
      респауна — или резерв-атрибут «однажды HP опустилось до 0
      от этого источника» без физической смерти?
    - **Q8.3.** Что происходит при «одной смерти»: отключение
      урона от этого источника + +X к ATTACK_DAMAGE (эволюция),
      или только иммун без бонуса?
    - **Q8.4.** Сохраняется ли список адаптаций через смерть-смерть-смерть
      (постоянная), или сбрасывается на де-трансформации (снятии костюма)?
    - **Q8.5.** Видимый UI / лог / звук на каждое «приобретение иммунитета»?

- **Q9. Эволюция фаз.** В комиксах Doomsday растёт после каждой
  «реинкарнации» (видимые шипы на разных стадиях). Делаем ли визуально
  меняющуюся модель (phase 1 / phase 2 / phase 3) — и если да, по
  какому триггеру?
  - По числу адаптаций?
  - По числу убитых противников?
  - По времени трансформации?

- **Q10. Регенерация.** Нужна ли пассивная регенерация HP (эффект
  Regeneration N как у Хоумлендера), и какого уровня?

- **Q11. Базовая иммунка.** Иммун к огню? Лаве? Утоплению?

### Способности (как у других героев)

- **Q12. Сколько активных способностей?** У других героев их 3–5.
  Возможные канон-механики:
  - **Smash (R / прыжок-слэм)** — приземление с шоквейвом и AoE
    урон (стиль Hulk Super Jump из Регулуса).
  - **Roar / Battle Cry** — AoE knockback + Weakness врагам.
  - **Berserk** — toggle-форма (повышенный урон / скорость, пониженный
    armor).
  - **Bone Spike** — выпуск шипа-снаряда (физический «дальний» удар).
  - **Adapt Force** — активная фаза «насильственной» адаптации (накапливает
    дополнительный иммунитет за счёт self-damage).

  Спросить пользователя поимённо — какие из них хочет, или другие.

- **Q13. Костюм.** Как трансформация — какой предмет, рецепт,
  rarity, slot? Аналогично существующим (`shadow_monarchs_cloak`,
  `homelander_suit`, `iron_man_armor`, `regulus_garb`).

### Ассеты

- **Q16. Иконка предмета.** Сделать face-extract как для Сон Джи Ву

### Прочее

- **Q17. Версия.** Какая? `v1.0.42`? `v1.1.0` (mejor bump)?
- **Q18. Локализация.** Только en + ru или ещё?
- **Q19. Совместимость.** Добавить отдельный PR или совмещать с
  фиксами других героев?

## 3. Что уже понятно по проекту (можно НЕ переспрашивать)

Эти пункты следуют из существующей инфраструктуры мода и
менять их не нужно:

- Hero ID будет `superheroes:doomsday`, package
  `com.example.superheroes.hero.DoomsdayHero`.
- Регистрация: `Heroes.java` + `HeroAttributes.java` + `AbilityIds.java`
  + `AbilityRegistry.java`.
- Размер: меняется через `Hero#getDimensions(Pose)` + Mojang-mixin
  `PlayerDimensionsMixin` (уже стоит в `superheroes.mixins.json`).
- Скины: `assets/superheroes/textures/entity/hero/doomsday.png`,
  накладывается через `HeroSkinLayer` (уже работает).
- Item: extends `TransformationItem`, регистрация в `ModItems.java` +
  creative tab `ModItemGroups.java`.
- Локализация: `en_us.json` + `ru_ru.json` (всегда обе).
- Networking: новых payload не нужно, если механика не требует
  HUD-индикатора. Если нужен — `DoomsdayAdaptationsS2CPayload`.
- Сборка: `./gradlew build --no-daemon -x test`. CI **не запускать**.

## 5. Рекомендованный архитектурный набросок (черновик, после ответов)

Эту секцию следующая сессия может использовать как стартовую точку,
но **ничего из неё не считать утверждённым** до подтверждения
пользователем.

```
hero/DoomsdayHero.java
  - getId() → superheroes:doomsday
  - getDimensions(Pose) → { Q1 }
  - applyPassives(player):
      add Attributes.MAX_HEALTH ≈ +Q3
      add Attributes.ARMOR ≈ +Q4.armor
      add Attributes.ARMOR_TOUGHNESS ≈ +Q4.toughness
      add Attributes.KNOCKBACK_RESISTANCE ≈ Q4.kbr
      add Attributes.ATTACK_DAMAGE ≈ +Q5
      add Attributes.MOVEMENT_SPEED ≈ +Q6
      Regeneration N → Q10
      Fire/Lava immune → Q11
  - cancelsFallDamage() → Q7

effect/DoomsdayAdaptationController.java   (если Q8 = да)
  - Map<UUID, Set<DamageType>> adaptations
  - на ALLOW_DAMAGE: если HP_after < 0 → запомнить тип, восстановить HP до 1,
    зарегистрировать иммун
  - на следующих ALLOW_DAMAGE с тем же типом → return false
  - очистка по Q8.4 (постоянная или per-transform)

ability/DoomsdaySmashAbility.java          (если Q12 включает smash)
ability/DoomsdayRoarAbility.java
ability/DoomsdayBerserkAbility.java
... (по списку Q12)

item/DoomsdaySuitItem.java                 (имя по Q13)

mixin: ничего нового, dimensions уже зашиты в PlayerDimensionsMixin

assets:
  textures/entity/hero/doomsday.png
  textures/item/<suit>.png
  lang/en_us.json + ru_ru.json
```
