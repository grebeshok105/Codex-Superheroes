# План v1.0.33 — Homelander overhaul + Infected Homelander Boss

Версия: **1.0.33** (или серия 1.0.33-1.0.36 если разнесём по этапам)
Скоуп: фикс регена, нерф брони, 3 новые способности игроку, реплика на босса, ребрендинг босса в "Infected Homelander".

## 0. Диагноз багов и решения

### 0.1 "Регенерация работает всегда, но в эффектах её нет"

**Корневая причина:** ванильный MC регенерирует HP естественно когда `foodLevel >= 18` и есть saturation. Наш `AutoSaturationController` (server tick каждые 40 тиков) даёт ВСЕМ игрокам с активным героем `SATURATION` amp 20 — то есть мгновенно набивает saturation до максимума. Это активирует ванильный saturation-regen (1 HP каждые 0.5 сек при saturation > 0).

В прошлом v1.0.31 я убрал из `HomelanderHero.applyPassives` бесконечный `Regeneration I`, и добавил `HomelanderRegenController` который выдаёт Regen I только при HP<20. Это сработало, иконки регена не видно. Но HP всё равно восстанавливается — потому что **ванильный saturation-regen продолжает работать в фоне**.

**Решение:**
- В `AutoSaturationController.tickPlayer`: пропускать Homelander-игроков. Либо вообще отключать saturation-buff для Homelander.
- Опционально: добавить мини-маркер «дикий герой» — ванильный голод не работает, saturation никогда не пополняется автоматически. Это значит Homelander не лечится естественно, но и не голодает.
- Чтоб полностью убить natural regen: можно повесить `MobEffects.HUNGER` amp 0 ambient/invisible/no-icon **либо** через mixin вернуть `false` в `Player.canEat()` для Homelander **либо** через tick-handler сбрасывать saturation в 0 каждый тик.
- **Выбираем простой путь**: `AutoSaturationController` пропускает Homelander, плюс `HomelanderRegenController` каждый тик ставит `foodData.setSaturation(0f)` чтоб ванильный regen-таймер не накапливался. Голод сам по себе медленный, не сломает геймплей.

### 0.2 "Босс не может убить, броня непробиваемая"

Сейчас Homelander имеет:
- ARMOR +35 (поле тушки 0 + 35 = 35 armor points = 80% mitigation)
- ARMOR_TOUGHNESS +6 (буст по толстым атакам)
- Resistance I (–20% урона) — постоянный ванильный effect
- Fire Resistance — постоянный
- HP +20 (max 40 HP)
- KB resistance 1.0

Это слишком жирный пресет. Эффективная HP против обычного hit (skip toughness) ≈ 40 / (1 - 0.80) / (1 - 0.20) = 250 HP. С toughness ещё больше.

**Решение (целевые значения):**
- ARMOR: 35 → **20** (сравнимо с full netherite armor: 8 cap + бонус — точнее netherite full = 20 armor = 80% mitigation, базовый netherite = 16 armor; делаем 20 как chestplate-tier polished)
- ARMOR_TOUGHNESS: 6 → **8** (netherite full = 12 toughness, чуть слабее)
- Убрать постоянный `DAMAGE_RESISTANCE` (Resistance I) из passives — это double-up с armor. Resistance оставим только для специальных моментов (если понадобится).
- HP +20 → оставляем (40 max, terrasic)
- Fire resistance — оставляем (фишка персонажа)
- KB resistance — оставляем

С такими статами эффективная HP ≈ 40 / 0.2 = 200 HP без resistance, что более-менее. Босс с лазером ~15/тик и 30 тик ствола = ~450 урона за один beam — должно убивать с 1.5-2 заходов.

### 0.3 "Регенерация даже после фикса"

После применения 0.1 (отключить auto-saturation для Homelander) — natural regen перестанет работать, а `HomelanderRegenController` всё ещё работает только при HP<20. Тогда:
- HP > 20: никакого регена, никакого восстановления
- HP < 20: Regeneration I, восстанавливается до full
- При full HP: Regen снимается до следующего падения ниже 20

## 1. Способности Homelander игрока — расширение

Сейчас: `flight`, `eye_lasers`, `x_ray` (3 способности).

Добавляем:

### 1.1 `stunning_roar` (Active, Energy)
- Cost: 30 energy
- CD: 8s
- Effect: AoE радиус 12 блоков, dmg 14 (raw, sound), screen distortion debuff на игроках в зоне 4с (без blindness — кастомный эффект "screen wave").
- VFX:
  - Server: `ParticleTypes.SONIC_BOOM` × 1 в сторону взгляда + `ParticleTypes.EXPLOSION` × 3 у рта
  - Sound: пользователь обещал прислать `homelander.roar.ogg`. Пока заглушка `WARDEN_SONIC_BOOM` + `RAVAGER_ROAR`.
  - Client overlay (новый эффект `screen_wave`): синусоидальная волна искажения экрана, без затемнения. Регистрируется через новый `ScreenWaveHud` или mixin в `GameRenderer.render` с post-process.
- Implementation:
  - `StunningRoarAbility` extends `Ability` (active)
  - `ScreenWaveEffect` (новый MobEffect, кастомный) — duration carrier; client рендер не трогает геймплей.
  - `ScreenWaveHud` подписан на overlay; рендерит вертикальные/горизонтальные сдвиги UV если эффект активен.

### 1.2 `hand_clap` (Active, Energy)
- Cost: 50 energy
- CD: 12s
- Effect: 90°-конус впереди радиус 18 блоков, dmg 20, knockback strong (vec.scale(3.5)).
- VFX:
  - Server: `ParticleTypes.CLOUD` массив (50) расширяющийся конусом + `ParticleTypes.EXPLOSION` × 2 в центре
  - Sound: `LIGHTNING_BOLT_THUNDER` 1.5x volume + `GENERIC_EXPLODE` low pitch
  - Client beam-like эффект (не лазер, а серая ударная волна). Реюзаем `BeamRenderer` с новым ID (`HandClapPayload`).
- Implementation:
  - `HandClapAbility` extends `Ability` (active)
  - `HandClapPayload` (S2C) → клиент рисует pink-ish ударную волну ~0.5 сек.
  - Server hit: ray-cast + `ShockwaveUtil.detonate` адаптация для конуса.

### 1.3 `iron_fists` (Toggle, Energy)
- Cost: 0 energy on toggle, 0.5/tick (10/s) drain
- CD: 0
- While active:
  - Visual: на модельке игрока появляются "железные кулаки" (overlay item на руках или цвет рук тёмно-серый). Через `HeroSkinLayer`-extension либо через `HumanoidArm` overlay в `PlayerRenderer`.
  - Auto-pull: каждые 5 тиков ищет ближайшего hostile-моба или target в радиусе 8 блоков, тянет его к себе вектором `(player.pos - target.pos).normalize().scale(0.4)`.
  - On melee hit (LivingHurt event server-side, while iron_fists active):
    - Damage cap 5 raw (вместо vanilla attack damage)
    - Strong knockback (`(player.lookAngle).scale(2.5)`)
    - Heavy screen shake на target client (через payload + client `ScreenShakeManager`)
- Implementation:
  - `IronFistsAbility` extends `Ability` (toggle)
  - `IronFistsController` server tick: пока toggle=on → tick auto-pull
  - `LivingEntityAttackMixin` (или Fabric `ServerLivingEntityEvents.ALLOW_DAMAGE` + replace) для override урона на 5
  - Server payload `ScreenShakeS2CPayload` (если такого нет)
  - Visual: extend `HeroSkinLayer` или новый `IronFistsLayer` накладывающий тёмные текстуры на `LEFT_HAND` / `RIGHT_HAND` модельные части. Через `PlayerRenderer.renderRightHand` mixin не лучший путь; используем feature renderer.

## 2. Дублирование на босса (Infected Homelander)

Сейчас у босса: `EyeLaserGoal` (1-в-1 с player), `HeatVisionSweep`, `LightningCall`, `ShockwaveDive`, `SonicSlam`, `Roar` (есть), `GroundMagnet` (есть), `BlockThrow` (есть), мили.

**Добавляем зеркально:**

### 2.1 `HomelanderBossStunningRoarGoal`
Заменяет / усиливает существующий `HomelanderRoarGoal`:
- Кастомный звук + screen distortion debuff на игроках в радиусе
- AoE 16 блоков, dmg 18

### 2.2 `HomelanderBossHandClapGoal`
- Telegraph 18 тиков (босс поднимает руки, искры)
- Forward конус 18 блоков 30° wide → dmg 24 + far knockback
- CD 9s

### 2.3 `HomelanderBossIronFistsGoal`
- Активный «iron fists» режим у босса (флаг на сущности `IRON_FISTS_ACTIVE`)
- При активации: 6с длительности
- Авто-pull одного игрока + телепорт к нему (rush-attack)
- Каждый mili-удар наносит 5 + сильно толкает + posts ScreenShake payload
- CD 25s

### 2.4 `X-Ray для босса` — пропускаем
X-Ray бесполезен для AI (mob уже видит target через стены через `getTarget()`).

### 2.5 `flight` — boss всегда летает (уже сейчас)
Не нужен toggle goal.

### 2.6 Уникальные для босса остаются: GroundMagnet, BlockThrow, отчасти LightningCall.
